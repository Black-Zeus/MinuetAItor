===============================================================================
FUENTES DE INFORMACION Y ORDEN DE LECTURA
===============================================================================

PASO 1 — Leer COMPLETAMENTE todos los archivos adjuntos antes de generar output.

El archivo de transcripción es la fuente primaria de contenido.
El archivo de resumen es la fuente secundaria; úsalo para:
  - Validar y completar lo extraído de la transcripción
  - Identificar capítulos y temas estructurados
  - Extraer puntos de acción listados explícitamente

El campo INPUT (METADATA DE LA SOLICITUD) contiene ÚNICAMENTE metadata
administrativa: participantes, fechas, cliente, perfil. No es el contenido
de la reunión. No uses el INPUT como fuente de contenido para scope,
agreements ni requirements.

PASO 2 — Extraer todos los temas identificados en los archivos.

Si el resumen incluye una sección "Capítulos y Temas", cada capítulo debe
originar al menos una sección topic en scope.sections.

Si la transcripción o el resumen incluyen "Puntos de Acción", cada punto
debe originar un item en agreements.items o requirements.items según
corresponda.

PASO 3 — Generar el JSON completo con todo lo extraído.


===============================================================================
REGLAS DE EXTRACCION PARA SCOPE
===============================================================================

Genera UNA sección por cada tema sustantivo identificado en los archivos.

MÍNIMOS OBLIGATORIOS:

• 1 sección introduction (siempre primera)
• 1 sección topic por cada capítulo o tema identificado en los archivos
  (mínimo 5 si la reunión dura más de 20 minutos)

PROCESO:

1. Leer la sección "Capítulos y Temas" del resumen si existe.
   Cada capítulo listado = una sección topic independiente.

2. Si no hay sección de capítulos, identificar cambios de tema en la
   transcripción. Cada cambio de tema = una sección topic independiente.

3. Para cada sección topic, extraer TODOS los bullets y sub-puntos como
   items en content.details.

NO agrupar múltiples capítulos en una sola sección.
NO omitir capítulos por considerarlos "poco relevantes".
El perfil orienta el énfasis, no autoriza omitir temas reales.


===============================================================================
REGLAS DE EXTRACCION PARA AGREEMENTS Y REQUIREMENTS
===============================================================================

Si los archivos incluyen "Puntos de Acción" o lista de compromisos:

• Cada punto de acción con responsable asignado → agreements.items
• Cada requerimiento técnico pendiente de una entidad → requirements.items

Si no hay responsable explícito → responsible: ""
Si no hay fecha → dueDate: null

No inventar responsables. No omitir puntos de acción listados.


===============================================================================
CONSISTENCIA DE inputInfo
===============================================================================

inputInfo es INMUTABLE.

Copiar EXACTAMENTE desde el INPUT recibido (METADATA DE LA SOLICITUD):

• attachments → desde INPUT.attachments (fileName, mimeType, sha256)
• meetingSchedule → desde INPUT.meetingInfo
• participants.attendees → desde INPUT.participants.attendees
• participants.copyRecipients → desde INPUT.participants.copyRecipients
• projectInfo.client → desde INPUT.projectInfo.client
• projectInfo.project → desde INPUT.projectInfo.project
• projectInfo.minuteTitle → desde INPUT.meetingInfo.title
• profileInfo → desde INPUT.profileInfo

Si un campo no existe en INPUT, usar el valor vacío del tipo correspondiente.

NUNCA inventar valores para inputInfo.
NUNCA copiar datos de los archivos adjuntos hacia inputInfo.

profileInfo debe copiarse desde inputInfo.profileInfo.
